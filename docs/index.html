Launching MiniCraft...

CheerpJ is downloading the JVM runtime (~30 MB) and executing the original MiniCraft.jar.
This takes 20–60 seconds on first visit. Files (textures, sounds, fonts) load from the same server.
If it fails, use the desktop version — CheerpJ doesn't support all Java APIs perfectly.

Failed to start

CheerpJ could not run MiniCraft. Common causes:

  • Your browser blocked the 30 MB JVM download
  • MiniCraft uses Java APIs that CheerpJ doesn't fully support (sound, fonts, threads)
  • A file failed to load from /app/

Use the desktop version instead:

Linux: ./run.sh  |  Windows: run.bat or java -jar MiniCraft.jar 







    
    








    
How this works: CheerpJ is a Java-to-JavaScript runtime that executes standard JAR files
    in the browser with zero code changes. The MiniCraft.jar here is the exact same JAR built from
    src/MiniCraft.java via javac.

    
Note: Browser audio requires a user click first. For multiplayer / voice chat, use the desktop ./run.sh.
